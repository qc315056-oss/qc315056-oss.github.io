前言

  • 在使用Vue3框架的页面程序中，至少要创建一个Vue实例，语法为Vue.createApp().mount()，这个Vue实例就相当于MVVM模式中的ViewModel。
  • 在创建Vue实例时，需要传递一个选项对象。
  • Vue 开发可以按两种不同的风格书写：选项式 API 和组合式 API（初学使用选项式，开发大型项目使用组合式）

  • 在vue中一个组件可能会被其他的组件引用，为了防止多个组件实例对象之间共用一个data，产生数据污染，于是将data定义成一个函数，使用return包裹，这样数据只在当前组件中生效，每个实例相互独立，组件之间不会相互影响。

计算属性的基本使用 01 定义： 计算属性是以函数形式，在Vue实例的选项对象的computed选项中定义。 计算属性的get和set  02 使用场景 get 和 set 的组合通常用于需要对计算属性进行双向绑定的场景。例如，当需要通过用户输入来更新依赖于多个数据源的计算属性时，set 方法可以用于解析用户的输入，并更新底层的数据状态。 注意事项 get 方法必须返回一个值，该值将成为计算属性的当前值。 set 方法接收一个参数，即新的值，可以在该方法中执行任何逻辑来更新相关的数据状态。 如果没有提供 set 方法，计算属性将被视为只读，尝试对其进行赋值不会生效。 因此在 computed 中可以分成 getter（读取） 和 setter（设值）。Vue 计算属性默认只有 getter**，常常省略不写。 setter：设置值时触发 getter：获取值时触发 当依赖的属性值发生变化时，计算属性的值会自动更新。不能通过getter直接修改计算属性，可以使用setter来执行。 应用：v-for和v-if一起使用的替代方案   03 例：

1

<div    v-for="(item,index) in MsgList"    :key="item.id"     v-if="index < 2"></div>

同时使用v-if和v-for，那么会根据整个数组生成所有组件，挂载完毕后才判断v-if，非常消耗资源。 在Vue3中，建议尽量避免在同一个元素上同时使用v-for和v-if，v-for 具有比 v-if 更高的优先级，这意味着 v-if 将分别重复运行于每个 v-for 循环中，将会影响运行性能，尤其是需要按需渲染的时候，所以不推荐v-if和v-for同时使用。如果确实需要根据条件过滤列表项并渲染，可以使用计算属性或方法来预处理数据。

watch状态监听：04

背景：Vue中的事件处理方法是根据用户所需自行定义的，不能自动监听当前Vue对象的状态变化。 定义：监听当前Vue实例中的数据变化，就会调用当前数据所绑定的事件处理方法。 参数：（newVal oldVal），监听的过程中可以拿到变化后的值和变化前的值 前提： Vue为了观察和响应Vue实例上的数据变动，提供了更通用的方式来监听属性——watch。 例：01 注意：监听器函数不能使用箭头函数定义（一旦使用箭头函数，使用this得到的值都是undefined） 应用场景： 在数据变化时执行异步或执行耗能很大的操作时。 监听器的形式： （1）深度监听：监听一个对象的属性变化 第一种：监听整个对象，对象里的每个属性值的变化都会执行handler   02 语法：  watch: {     user: {       handler(val){  // 注意：参数不再分newVal和oldVal，只能取到最新的值         console.log(val);       },       deep: true     }   } handler用于定义当数据变化时调用的监听器函数 deep用于监听对象属性变化，选项值为true，表示无论对象的属性在对象中层级有多深，只要属性值发生变化，都会被监听到。 注意： handler: 固定方法触发。侦听函数必须叫handler(必写) deep: 开启深度侦听(必写) 第二种：监听对象的某个属性，被监听的属性值发生变化就会执行函数   03 当对象属性过多时，使用deep属性进行深度监听会给所有的属性加上监听器, 如果只是想对其中一个属性进行监听， 那么可以直接使用字符串形式的监听。 语法： watch: {     ‘user.name’: function( newVal, oldVal ){ console.log(newVal, oldVal);   // 注意：函数执行后，获取的 newVal 值和 oldVal 值不一样 } } （2）即时监听  04 watch 默认是懒执行（仅当数据源变化时才会执行），所以在初始渲染时并不会被调用，只有在后续监听的属性发生 变化时才会被调用，如果要让监听器函数在监听开始后立即执行，可以使用immediate : true选项。 （3）一次性监听  05 希望回调只在源变化时触发一次，请使用 once: true 选项。

数组变动更新检测

对于数组，Vue 3.0 提供了一些特殊的变更方法，这些方法会确保视图在数组内容发生变化时得到更新。这些变更方法包括： 数组的变异方法：（操作数组改变原数组） push（）往数组最后面添加一个元素，成功返回当前数组的长度
pop（）删除数组的最后一个元素，成功返回删除元素的值
shift（）删除数组的第一个元素，返回该元素。
unshift（）在数组的开头添加一个或者多个元素，返回原数组的长度，改变原来的数组
splice（）有三个参数，第一个是删除index位置的元素（必选），第二个是想要删除的个数（必选），第三个是删除后想要在原位置替换的值（可选）
sort（）使数组按照字符编码默认从小到大排序,成功返回排序后的数组
reverse（） 将数组倒序，成功返回倒序后的数组 数组的非变异方法：（操作数组不改变原数组，生成新数组） filter（）可以对数组元素进行过滤 concat（）将一个或多个数组组合到原数组中，组成新的数组返回 slice（startindex，endindex）从startindex下标开始向后截取到endindex位置（不包含）的元素，截取的这些元素组成新的数组返回，原数组不变 总结： vue能不能检测到数组的变化并更新，取决于原生js的数组方法。 如果原生js方法能够修改原数组（vue变异方法），那么vue就可以检测到变化并更新 如果原生js方法不能够修改原数组（vue非变异方法），而是返回一个新数组，那么vue也可以检测到变化并更新，前提是使用这些方法时要把返回的新数组替换掉旧数组。

前提： 内置指令可以满足大多业务需求，但如果我们需要对普通DOM元素进行底层操作，要用到自定义指令。 提示： 自定义指令需要注册后才能使用

Vue提供了两种注册方式：全局注册和局部注册

钩子函数  02             

自定义指令的功能（第二个参数）由钩子函数组成，Vue为我们提供了自定义指令的钩子函数： beforeMount：指令绑定元素-挂载前 mounted：指令绑定元素-挂载后  beforeUpdate：指令绑定元素数据修改-触发修改前 updated：指令绑定元素数据修改-触发修改后 beforeUnmount：指令绑定元素-销毁前 unmounted：指令绑定元素-销毁后

钩子函数可以携带的参数：  03

（1）el：指令绑定的元素。这可以用于直接操作 DOM。 （2）binding：一个对象，包含以下 键值对： value：指令的绑定值，例：v-dir=“1 + 1"中，绑定值为 2 oldValue：指令绑定的前一个值，仅在 beforeUpdate 和 updated 钩子中可用。无论值是否改变都可用。 arg：传给指令的参数 (如果有的话)。例：v-dir: foo中，参数是 “foo” modifiers：一个包含修饰符的对象 (如果有的话)。 例：v-dir.abc.abd中，修饰符对象为 { abc: true, abd:true } instance：使用该指令的组件实例。 dir：指令的定义对象。 （3）vnode：代表绑定元素的底层虚拟节点 （4）preVnode：代表之前的渲染中指令所绑定元素的虚拟节点，仅在beforeUpdate 和 updated钩子中使用。 注意：除了el之外，其他参数就是只读参数，不能修改。

动态指令参数  04

==vue3中任何动态的都要用方括号表示== 格式：自定义指令[argument]=‘value’ 函数简写 自定义指令在mounted 和 updated钩子函数上实现相同行为，且只需要用到这两个函数，那么可以直接用一个函数来定义指令 例： Vue.createApp({}).directive(‘color’, (el, binding) => {     el.style.color= binding.value;  // 这在 ‘mounted’ 和 ‘updated’ 时都调用 }) 对象字面量 如果指令需要多个值，可以传入一个JS对象字面量。 注意：指令可以接收所有合法的JS表达式。 例：

Vue.createApp({}).directive('demo', (el, binding) => {     console.log(binding.value.color)  //white console.log(binding.value.text)   //hello })

Vue组件的基本概念

Vue有一个非常重要的特性：组件化开发，在开发过程中使用非常频繁而灵活。 定义： 组件是构成页面的独立结构单元。我们只需按照Vue规范定义组件，将组件渲染到页面，善用组件能够实现复杂的页面结构，降低代码之间的耦合程度，提高代码的可复用性。 组件名大小写 定义组件名的方式有两种： 如果组件是在本页面注册且直接使用时，只能使用 kebab-case (短横线分隔命名) 例：

特点：字母全小写且必须包含一个连字符 如果组件通过导入形式在页面中使用时，使用 PascalCase (驼峰式命名) 例：

组件之间的数据传递

由于组件具有自己的局部作用域，所以组件之间的数据传递需要借助一些工具来实现。 父组件与子组件之间的依赖关系： 父组件向子组件传递是数据从外部向内部传递（父组件通过props向下传递数据给子组件） 子组件向父组件传递是数据从内部向外部传递（子组件通过events给父组件发送消息）

在组件中使用prop传递数据

在Vue中，数据传递主要通过props属性和 $emit方式来实现 props( 父组件–>子组件 ) 在组件中，props用来声明需要从父级接收的数据 利用props属性传值：array(数组)  object(对象)（04-5见， 04-7讲解） 注意： （1）组件之间常用的传值类型：数组  04-1 （2）由于HTML特性不区分大小写，在DOM中使用时，驼峰命名的props名称要转为短横分隔命名 04-2 （3）有时候，传递的数据并不是直接静态的，而是来自父级的动态数据。这时可以使用指令v-bind来动态绑定 props 值，当父组件的数据变化时，也会传递给子组件。  04-3 （4）如果你要直接传递数字、布尔值、数组、对象而不使用v-bind，那么传递的内容是单纯的字符串  04-4

单向数据流

数据从父级组件传递给子组件，只能单向绑定。子组件内部不能直接修改从父级传递过来的数据。 我们经常会遇到两种需要改变prop的情况： （场景1）父组件传递初始值进来，子组件将它作为初始值保存起来，在自己的作用域下可以随意使用和修改。 解决方法：在组件 data 再声明一个数据，引用父组件的 prop  04-5 （场景2）prop 作为需要被改变（传递的原始值不能直接使用，需要有改动）的原始值传入 解决方法：计算属性  04-6
