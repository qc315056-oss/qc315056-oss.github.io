let和const作用域

let关键字

  • 作用：声明变量
  • 特点：所声明的变量，只在let命令所在的代码块内（块级作用域）有效

const关键字

  • 作用：声明一个只读的常量
  • 特点：一旦声明，常量的值就不能改变
  • 注意：const一旦声明变量，就必须立即初始化，不能留到以后赋值

var关键字的几个特点

  1. 存在变量提升 （变量提升：变量可以在声明之前使用，值为undefined）
  2. 没有暂时性死区 （暂时性死区：let/const命令会使区块形成封闭的作用域，若在声明之前使用变量，就会报错）
  3. 可以重复声明

let const和var之间的区别

  1. 创建了块级作用域 let和const具有块级作用域 var不存在块级作用域
  2. 变量提升 var存在变量提升 let和const不存在变量提升，即在变量只能在声明之后使用，否则会报错
  3. 添加属性 浏览器的全局对象是window，Node的全局对象是global var是声明的变量为全局变量，并且会将该变量添加为全局对象的属性，但是let和const不会
  4. 重复声明、 var声明变量时，可以重复声明变量，后声明的同名变量会覆盖之前声明的变量 const和let不允许重复声明变量
  5. 暂时性死区 使用let、const命令声明变量之前，该变量都是不可用的 使用var声明的变量不存在暂时性死区
  6. 初始值设置 在变量声明时，var和let可以不用设置初始值 const声明变量必须设置初始值
  7. 指针指向 let创建的变量是可以更改指针指向（即可以重新赋值） 但const声明的变量时不允许改变指针的指向

局部作用域

局部作用域分为块级作用域和函数作用域

块级作用域

块级作用域解决了js中的两个问题： 1. 内层变量可能覆盖外层变量 2. 用来计数的循环变量泄露为全局变量 概念：任何一对花括号中的语句集多属于一个块，在这之中定义的所有变量在代码块外都是不可见的 注意：

  1. ES6允许块级作用域的任意嵌套。内层作用域可以定义外层作用域的同名变量
  2. 块级作用域必须有大括号，如果没有大括号，JS引擎就认为不存在块级作用域

函数作用域

在函数内部声明的变量只能在函数内部被访问，外部无法直接访问

1
2
3
4
5

function getSum(){
	//函数内部是函数作用域 属于局部作用域
	const num = 10
}
console.log(num) //此处报错 函数外部不能使用局部作用域变量

总结：

  1. 函数内部声明的变量，在函数外部无法被访问
  2. 函数的参数也是函数背部的局部变量
  3. 不同函数内部声明的变量无法互相访问
  4. 函数执行完毕后，函数内部的变量实际被清空了

全局作用域

<script>和.js文件的「最外层」就是所谓的全局作用域，在此声明的变量在函数内部也可以被访问 全局作用域中声明的变量，任何其他作用域都可以被访问

  3. 尽可能少地声明全局变量，防止全局变量被污染

变量的解构赋值

数组解构

数组解构是将数组的单元值快速批量赋值给一系列变量的简洁语法 基本语法： 1. 赋值运算符 = 左侧的[]用于批量声明变量，右侧数组的单元值将被赋值给左侧的变量 2. 变量的顺序对应数组单元值的位置一次进行赋值操作

let b = 3;
[b,a] = [a,b]
console.log(a) //3
console.log(b) //1

对象解构

对象解构是将对象属性和方法快速批量赋值黑一系列变量的简洁语法 基本语法： 1. 赋值运算符=左侧的{}用于批量声明变量，右侧对象的属性值将赋值给左侧的变量 2. 对象属性的值将被赋值给与属性名相同的变量 3. 注意解构的变量名不要和外面的变量名冲突否则报错 4. 对象中找不到与变量名一致的属性时变量值为undefined 要求：属性名和变量名必须一致

  4. 可以使用replace()结合【正则表达式】实现代码效果同时对齐
  5. \n\表示换行

应用

  1. $()中可以放 任意js表达式进行运算 引用对象属性 调用函数

函数的扩展

函数参数的默认值

ES6为函数的参数设置默认值，将默认值直接写在参数定义的后面 注意：

  • 参数变量是默认声明的，所以不能用let或const再次声明，但用var可以
  • 使用参数默认值时，函数不能有同名参数 应用：
  1. 与结构赋值默认值结合使用，称为双重默认值。
  2. 将需要设置默认值的参数放在尾部。因为非尾部的参数设置默认值，参数不可以省略传递
  3. 参数指定默认值值后，函数的length属性返回没有指定默认值的参数个数 作用域： 函数设置了参数的默认值，参数会形成一个单独的作用域，等到执行结束，这个作用域就会消失。这种语法行为，在不设置参数默认值时不会出现

rest参数

形式：…变量名，用于获取函数的多于参数。变量名的形式为数组，将多余参数放入数组中 注意：

  1. rest参数之后不能再有其他参数（即只能是最后一个参数），否则会报错
  2. 函数的length属性，不包rest参数

箭头函数

形式：= > 注意：

  1. 若箭头函数只有一个参数，圆括号可以省略；没有参数或多个参数，圆括号不能省略
  2. 若箭头函数的执行体只有一条语句，大括号和return语句可以省略，执行体多余一条语句，大括号和return语句不能省略
  3. 若返回一个对象形式的返回结果，必须在对象外面加上括号，否则会报错 应用： 箭头函数可以与变量结构结合使用

箭头函数的this问题

  1. (important)箭头函数没有自己的this对象 箭头函数内部的this就是上层作用域中的this
  2. 箭头函数不可以结合new成为构造函数，否则报错，因为箭头函数内部没有this
  3. 箭头函数可以结合rest参数使用
  4. 不可以使用yield命令，因此箭头函数不能用作Generator函数
  5. 除了this，以下三个变量在箭头函数之中也是不存在的，指向外层函数的对应变量：arguments、super、new.target
  6. 由于箭头函数没有自己的this，所以当然也就不能用call()、apply()、bind()这些方法去改变this的指向

数值的扩展

知识点回忆

在JavaScript中，以下针对于数值型的方法有怎样的作用？ isFinite() isNaN() parseInt() parseFloat() 在ES6中，以上四个方法为： Number.isFinite() Number.isNaN() Number.parseInt() Number.parseFloat()

JS全局属性和方法：https://www.runoob.com/jsref/jsref-obj-global.html JSNumber对象的属性和方法：https://www.runoob.com/jsref/jsref-obj-number.html

以上四个函数移植到Number对象的目的： 慢慢地减少全局性的函数，把全局函数合理地规划到其他对象下，渐渐实现语言的模块化

isNaN()与Number.isNaN()

  • isNaN方法接收一个参数x，如果参数x能被转换为数字，返回false；如果参数x不能转换为数字，则返回true
  • Number.isNaN()方法也是接收一个参数x，判断参数x是否为NaN这个值，如果是，则返回true，否则返回false 两个方法之间的区别： isNaN()方法并不是严格的判断传入的值是否等于NaN，只是将传入的值尝试转换为数值型，故isNaN() 函数会影响NaN的准确判断，如果想要判断一个值是否严格的等于NaN，推荐使用Number.isNaN()方法。

isFinite() 与 Number.isFinite()

isFinite() 方法接收一个参数x，先尝试将参数转换为数值型，如果转换的结果是NaN 这个值、正无穷大或负无穷大，返回 false，其他值返回 true。 Number.isFinite()方法也是接收一个参数x，判断参数x是否为数值型，如果是，则返回true；否则返回false；

Number.parseInt(),Number.parseFloat()

ES6 将以上两个全局方法移植到Number对象上面，行为完全保持不变。

数组的扩展

扩展运算符的应用

  1. 将数组转化为用逗号分隔的参数列表（[1,2]→1，2），一般用于函数调用（rest参数）
  2. 复制数组 数组是复杂数据类型，直接复制只是复制了对数组的引用，而不是复制产生一个新数组
  3. 合并数组 若对基本数据类型的变量进行复制，得到两个值相同的变量，改变其中一个变量的值，不会影响另一个变量的值；但是对引用数据类型（数组、对象）的变量指向同一个对象的情况，改变其中一个变量的内容，另一个变量也会发生改变，这就是浅拷贝。与浅拷贝相对的是深拷贝，深拷贝是真正创建一个对象的副本。[[深拷贝与浅拷贝-随笔]]

    浅拷贝的三种实现方式：

    1. 对象方法Object.assign()（只能对象数据）
    2. …（既可以对象数据、又可以数组数据）

定义： Set是一个构造函数，用来生成 Set 数据结构。 Set数据结构是一个类似数组的对象，与数组的区别是成员的值唯一，不能重复。 参数： 具有遍历器接口的数据结构（数组、字符串、类似数组的对象） 应用： 数组去重、字符串去重（案例） 实例属性： size：获取Set实例的成员总数。 操作方法（用于操作数据） add()：添加某个值，返回 Set 结构本身 添加值时遵循ES6“同值相等”算法，对以下情况做特殊处理：

  • 添加两个NaN，结果只会添加1个，因为NaN===NaN  //true
  • 添加两个{}，结果为2个，因为对象是独立唯一的
  • 添加两个{“id”:“2”}，结果为2个，因为对象是独立唯一的 delete()：删除某个值，返回一个布尔值，表示删除是否成功 has()：判断值是否为Set的成员，返回一个布尔值 clear()：清除所有成员，没有返回值 遍历方法（用于遍历成员） keys()：遍历键名 values()：遍历键值 entries()：遍历键值对 注意：

Symbol

前提

js的对象属性名都是字符串，这容易造成属性名的冲突。 为了从根本上防止属性名的冲突，于是ES6 引入了Symbol。

Symbol

  • Symbol表示独一无二的值，它是一种新的数据类型。 数据类型：String、Number、Boolean、undefined、null、Object、Symbol
  • Symbol 值通过Symbol()函数生成 例：let s = Symbol(); typeof s   // “symbol”
  • Symbol()函数的参数表示对当前 Symbol 值的描述
  • Symbol数据类型不能直接与其他类型的值进行运算，会报错；
  • 如需转换类型，可以将Symbol转为字符型或逻辑型

Symbol的应用

  • 作为对象的属性名，就能保证不会出现同名的属性，防止某键被改写或覆盖。注意：Symbol 值作为对象属性名，要使用[ ]进行设置或访问而不是用点运算符，因为点运算符后面总是字符串，所以不会读取正确的Symbol值

Symbol的遍历

  • 如果Symbol 值作为属性名，遍历对象的时候，Symbol属性会被for-in、for-of循环忽略掉。 重点：for-in和 for-of的区别？ for-in 适用于遍历普通对象、数组 for-of 适用于遍历可迭代对象（数组、字符串、Map、Set等）（推荐！）
  • Object.getOwnPropertySymbols() 可以获取指定对象的所有 Symbol 属性名 该方法返回一个数组，成员是当前对象的所有用作属性名的 Symbol 值

lterator（遍历器）

遍历器（Iterator）是一种功能，为各种不同的数据结构提供统一的访问机制。数据结构只要部署 Iterator 接口，就可以完成遍历操作。 有些数据结构原生具备Iterator接口，不用任何处理就可以被for…of循环遍历

Promise对象

前提

JS当中有两种编程思想：同步、异步。 JS语言的执行环境是“单线程”的（脑子一根筋，做着当前的这件事情，没有完成之前，绝对不会做下一件事情）

同步与异步

定义： 同步：任务顺序执行，上一个任务执行完毕才会执行下一个方法。（JS中大部分都是同步编程） 异步：暂时搁置当前任务，等到主线程任务执行完成，再来响应该任务。 在JS中，异步只有四种情况： –> 定时器都是异步的 –> 所有的事件绑定都是异步的 –> Ajax（前后端交互）读取数据都是异步的 –> 回调函数都是异步的 例：

1
2
3
4
5
6

var n = 0;
window.setTimeout(function () {
    n++;
    console.log(n);    //后执行
},1000);
console.log(n);   //先执行

同步和异步区别：

  1. 同步是阻塞模式，异步是非阻塞模式。
  2. 同步发送一个请求，需要等待返回，然后才能发送下一个请求。有个等待过程；异步发 送一个请求，不需要等待返回，直接就可以发送下一个请求。

Promise的概念

背景： 

1
2
3
4
5

fs.readFile(fileA, 'utf-8', function (err, data) {
    fs.readFile(fileB, 'utf-8', function (err, data) {
      // ...
    });
});

如果依次读取两个以上的文件，就会出现多重嵌套。这时的代码由于多个异步操作发生形成了1强耦合，只要有一个操作需要修改，它的上层回调函数和下层回调函数，可能都要跟着修改。这种情况就称为"回调函数地狱"。Promise对象就是为了解决这个问题而设计的，解决方式是将回调函数的嵌套改成链式调用。

定义： 所谓Promise，简单说就是一个容器，Promise本身不是异步的，只是内部包含了异步操作的代码。从语法上说，Promise 是个对象，从中可以获取异步操作的消息。Promise 提供统一的 API，各种异步操作都可以用同样的方法进行处理。 特点：

  • 对象的状态不受外界影响 Promise对象代表一个异步操作，有三种状态：pending（进行中）、fulfilled（已成功）和rejected（已失败）。只有异步操作的结果，可以决定当前是哪一种状态，任何其他操作都无法改变这个状态。
  • 一旦状态改变会一直保持，任何时候都可以得到这个结果 Promise对象的状态改变，只有两种可能： 从pending变为fulfilled 从pending变为rejected 只要这两种情况发生，状态会一直保持这个结果，这时就称为 resolved（已定型） 如果改变已经发生了，后续再对Promise对象进行回调，也会得到这个结果。 优缺点： 优点：
  • Promise对象可将异步操作以同步操作的流程表达出来，避免了层层嵌套的回调函数
  • Promise对象提供统一的接口，使得控制异步操作更加容易 缺点：
  • Promise无法取消，一旦新建它就会立即执行，无法中途取消
  • 如果不设置回调函数，Promise内部抛出的错误，不会反应到外部
  • 当处于pending状态时，无法得知目前进展到哪一个阶段（刚刚开始还是即将完成）

Promise的基本用法

ES6 规定，Promise对象是一个构造函数，用来生成Promise实例。 参数： 接受一个函数作为参数，该函数的两个参数分别是resolve函数和reject函数 resolve函数的作用： 将Promise对象的状态从“未完成”变为“成功”（即从 pending 变为 resolved），在异步操作成功时调用，并将异步操作的结果作为参数传递出去 reject函数的作用： 将Promise对象的状态从“未完成”变为“失败”（即从 pending 变为 rejected），在异步操作失败时调用，并将异步操作报出的错误作为参数传递出去 注意：

  • Promise 新建后就会立即执行。
  • 调用resolve或reject并不会终结 Promise 的参数函数的执行。

Promise的相关方法

Promise.prototype.then()

【作用】 Promise实例生成以后，用then方法分别指定resolved状态和rejected状态的回调函数。 【参数】 两个回调函数，第一个参数是resolved状态的回调函数，第二个参数是rejected状态的回调函数。这两个函数都是可选的，并且都接受Promise对象传出的值作为参数 【返回】 返回一个新的Promise实例（因此可以采用链式写法，即then方法后面再调用另一个then方法）

Promise.prototype.catch()

是 .then(null, rejection) 或 .then(undefined, rejection) 的别名，用于指定发生错误时的回调函数。 【注意】

  • catch是一个语法糖，本质上还是通过then 来处理的。
  • then的第二个参数和catch捕获错误信息的时候使用就近原则，reject抛出错误后，then的第二个参数和catch方法都存在的情况下，只有then的第二个参数能捕获到，如果then的第二个参数不存在，则catch方法会捕获到。
  • 一般来说，不在then()方法里面定义Reject状态的回调函数，总是使用catch方法。

Promise.prototype.finally()

finally方法的回调函数不接受任何参数，用于指定不管 Promise 对象最后状态如何，都会执行的操作。

Promise.prototype.all()

将多个 Promise 实例，包装成一个新的 Promise 实例。 【格式】 const p = Promise.all([p1, p2, p3]); 【参数】 具有Iterator接口且每个成员都是Promise实例的数据结构。 【规则】

  • 只有p1、p2、p3的状态都变成fulfilled，p的状态才会变成fulfilled，此时p1、p2、p3的返回值组成一个数组，传递给p的回调函数。
  • 只要p1、p2、p3之中有一个被rejected，p的状态就变成rejected，此时第一个被reject的实例的返回值，会传递给p的回调函数。 【注意】 如果作为参数的 Promise 实例，自己定义了catch方法，那么它一旦被rejected，并不会触发 Promise.all() 的catch方法。

Promise.race() （极少用）

将多个 Promise 实例，包装成一个新的 Promise 实例。 【格式】 const p = Promise.race([p1, p2, p3]); 【参数】 具有Iterator接口且每个成员都是Promise实例的数据结构 【规则】 只要参数之中有一个实例率先改变状态，p的状态就跟着改变。那个率先改变的 Promise 实例的返回值，就传递给p的回调函数。

Promise.allSettled()（极少用）

将多个 Promise 实例，包装成一个新的 Promise 实例。 【格式】 const p = Promise.allSettled([p1, p2, p3]); 【参数】 具有Iterator接口且每个成员都是Promise实例的数据结构 【规则】 只有等到参数中的所有 Promise 对象都发生状态变更，（不管是fulfilled 还是rejected），返回的Promise对象才会发生状态变更，一旦发生状态变更，状态总是fulfilled。

Promise.any()（极少用）

将多个 Promise 实例，包装成一个新的 Promise 实例。 【格式】 const p = Promise.allSettled([p1, p2, p3]); 【参数】 具有Iterator接口且每个成员都是Promise实例的数据结构 【规则】 只要参数实例有一个变成fulfilled状态，返回的Promise对象就会变成fulfilled状态；如果所有参数实例都变成rejected状态，返回的Promise对象就会变成rejected状态。 【返回】 Promise.any()抛出的错误，不是一个一般的 Error 错误对象，而是一个AggregateError 实例。它相当于一个数组，每个成员对应一个被rejected的操作所抛出的错误。

Promise.resolve()

将现有对象转为状态是resolved的Promise 对象 应用场景：

  1. 参数是一个 Promise 实例 Promise.resolve()将不做任何修改、原封不动地返回这个实例。
  2. 参数是一个具有then方法的对象 Promise.resolve()会将这个对象转为 Promise 对象，然后就立即执行then()方法。
  3. 参数不是具有then()方法的对象，或根本就不是对象 Promise.resolve()方法返回一个新的 Promise 对象，状态为resolved。
  4. 不带有任何参数 Promise.resolve()方法直接返回一个resolved状态的 Promise 对象。

Promise.reject()

将现有对象转为状态是rejected的Promise 对象 应用场景： Promise.reject()方法的参数，会原封不动地作为reject的理由，变成后续方法的参数。

深拷贝与浅拷贝

深：你变我不变 浅：你变我也变

基本类型数据（数字、字符、布尔）——深拷贝 复杂类型数据（数组、对象）——浅拷贝

Object.assign()——对象的合并方法 concat()——数组的合并方法 原理：把一堆小的融合为一个大的。

Object.assign()执行浅拷贝的原理是： 将一个空对象和有内容的对象进行合并，达到复制的效果。 concat()执行浅拷贝的原理是： 将一个空数组和有内容的数组进行合并，达到复制的效果。 concat()和slice()任选一个来执行浅拷贝即可！

深拷贝——只针对数组和对象！！

深拷贝：数据操作处理 浅拷贝：数据引用
